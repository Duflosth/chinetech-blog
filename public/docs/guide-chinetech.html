Guide simple pour non-technicien

Gérer le blog CHINETECH

Ce guide explique les services, les comptes à créer, comment écrire des articles, et comment demander à Codex de modifier le site sans rentrer dans la technique.

Site public chinetech.netlify.app
Articles / CMS /admin/
Code / historique GitHub
La phrase à retenir : pour écrire un article, on utilise le CMS. Pour modifier le site lui-même, on demande à Codex de changer le code dans GitHub. Netlify publie ensuite automatiquement le résultat.

Les services

Voici les services à connaître. Pas besoin de tout maîtriser : il faut surtout savoir à quoi chacun sert.

Service Métaphore À quoi ça sert Compte à créer ?
GitHub L'armoire + la machine à remonter le temps. Stocke le projet et garde l'historique de chaque changement. Oui si on veut utiliser Codex sur le code ou collaborer sur le projet.
Netlify L'imprimeur automatique. Publie le site en ligne dès que GitHub change. Gère aussi les formulaires et les accès CMS. Oui pour le propriétaire. Non pour un simple rédacteur CMS.
Decap CMS Le formulaire de rédaction. Permet d'écrire des articles sans toucher au code. Non directement. On se connecte avec Netlify Identity.
Netlify Identity Le badge d'entrée. Autorise les personnes invitées à entrer dans le CMS. Oui, mais seulement via une invitation email.
Git Gateway Le coursier. Transporte les articles du CMS vers GitHub. Non. C'est une configuration côté Netlify.
Astro La machine de mise en page. Transforme les fichiers du projet en pages web rapides. Non.
Codex L'assistant qui sait modifier l'atelier. Aide à changer le code, corriger un bug, ajouter une page, modifier un style. Oui via ChatGPT/Codex, et souvent GitHub connecté.
Différence GitHub / Netlify : GitHub garde les fichiers. Netlify affiche le site au public. GitHub est la réserve, Netlify est la vitrine.

Quels comptes créer ?

Il ne faut pas créer tous les comptes pour tout le monde. Chaque personne doit avoir seulement ce dont elle a besoin.

Personne Compte recommandé Pourquoi
Propriétaire du projet GitHub + Netlify + ChatGPT/Codex Il gère le site, les accès, les déploiements et les demandes de modifications.
Rédacteur non technique Netlify Identity uniquement Il écrit dans le CMS. Il n'a pas besoin d'aller sur GitHub.
Personne qui va demander des changements à Codex GitHub + ChatGPT/Codex Codex doit pouvoir lire/modifier le projet dans GitHub.
Lecteur ou abonné newsletter Aucun compte Il lit le site et peut laisser son email.

Créer un compte GitHub

  1. Aller sur github.com.
  2. Cliquer sur Sign up.
  3. Choisir un email fiable.
  4. Choisir un nom d'utilisateur simple.
  5. Valider l'email reçu.
  6. Activer la double authentification si GitHub le demande.

Créer l'accès CMS d'un rédacteur

  1. Dans Netlify, aller sur le site CHINETECH.
  2. Ouvrir Identity.
  3. Cliquer sur Invite users.
  4. Entrer l'email du rédacteur.
  5. Le rédacteur clique sur le lien reçu et choisit son mot de passe.
  6. Il se connecte ensuite sur /admin/.
Important : un lien d'invitation ou de récupération est secret. Ne jamais le coller dans ChatGPT, Slack, WhatsApp ou un document partagé.

Git et GitHub expliqués

Git est le principe de suivi des changements. GitHub est le site qui héberge ce suivi.

Git

Git est comme un carnet de bord. À chaque changement important, on prend une photo de l'état du projet. Cette photo s'appelle un commit.

GitHub

GitHub est le bâtiment où l'on range ce carnet de bord. Il permet aussi à plusieurs personnes ou assistants de travailler sans perdre l'historique.

Les mots à connaître

Mot Version simple
Repository Le dossier du projet sur GitHub.
Commit Une sauvegarde nommée d'un changement.
Branch Une copie de travail pour tester sans abîmer la version principale.
Push Envoyer ses changements vers GitHub.
Pull request Demander une relecture avant d'intégrer un changement.
Markdown Une façon simple d'écrire du texte avec des titres, listes et liens.
Pour un non-technicien : il n'est pas nécessaire de manipuler Git. Le CMS et Codex peuvent faire le travail. Mais comprendre les mots aide à ne pas paniquer.

Écrire des articles

Le bon chemin pour écrire est le CMS : chinetech.netlify.app/admin/.

Créer un article

  1. Se connecter à /admin/.
  2. Ouvrir Articles.
  3. Cliquer sur New Article.
  4. Remplir le titre, la description, la date, l'auteur et les tags.
  5. Écrire le texte dans le champ principal.
  6. Laisser draft: true tant que l'article n'est pas prêt.
  7. Quand tout est relu, passer draft: false.
  8. Sauvegarder.
  9. Attendre que Netlify publie le site.

Les champs les plus importants

Champ À quoi ça sert
title Le titre de l'article.
description Le résumé court qui donne envie de lire.
tags Les catégories : Tech, IA, Business, Société, etc.
draft true = invisible. false = publié.
featured Mettre l'article dans la section "À la une".

Mini-recette d'un bon article

  • Un titre clair.
  • Une première phrase qui dit pourquoi le sujet compte.
  • Trois parties maximum si possible.
  • Des faits vérifiés.
  • Une conclusion simple : ce qu'il faut surveiller ensuite.

Modifier le site avec Codex

Codex sert à modifier le code du site : design, pages, formulaires, corrections, bugs. Il ne remplace pas le CMS pour écrire un article simple.

Deux façons d'utiliser Codex

Plus simple

Codex dans ChatGPT

On connecte GitHub, on choisit le projet, puis on demande une modification. C'est le plus adapté pour une personne peu technique.

Plus avancé

Codex dans le terminal

On travaille depuis l'ordinateur, dans le dossier du projet. C'est utile pour quelqu'un qui sait déjà ouvrir un terminal.

Quand utiliser Codex ?

  • Changer un texte présent dans la page d'accueil.
  • Ajouter une page statique.
  • Modifier le design.
  • Corriger un problème de lien ou d'affichage.
  • Ajouter un champ au CMS.
  • Générer un document HTML/PDF comme celui-ci.

Quand ne pas utiliser Codex ?

  • Pour écrire un article normal : utiliser le CMS.
  • Pour réinitialiser un mot de passe : utiliser Netlify Identity.
  • Pour envoyer un secret ou token : ne jamais le coller à un assistant.

Routine simple

Pour éviter les erreurs, il suffit de suivre quelques habitudes simples.

Pour un nouvel éditeur

  1. L'inviter dans Netlify Identity.
  2. Lui faire créer son mot de passe.
  3. Lui envoyer le lien /admin/.
  4. Lui demander de créer un article test en brouillon.
  5. Vérifier ensemble que le champ draft est compris.

Avant publication

  • Le titre est clair.
  • Le résumé est court.
  • Les faits importants sont vérifiés.
  • Les tags sont cohérents.
  • draft est passé à false seulement à la fin.
  • Le site est vérifié après le déploiement.

Problèmes fréquents

Problème Réponse simple
Le lien d'invitation ne marche pas Renvoyer une nouvelle invitation. Les anciens liens expirent ou se consomment.
Rate limit exceeded Attendre 30 à 60 minutes. Ne pas cliquer plusieurs fois.
L'article n'apparaît pas Vérifier draft, puis attendre le build Netlify.
Le site affiche 404 Vérifier l'orthographe : chinetech.netlify.app.
Le CMS ne publie pas Vérifier Identity et Git Gateway dans Netlify.
